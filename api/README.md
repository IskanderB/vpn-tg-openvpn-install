# OpenVPN User Management API

REST API для управления пользователями OpenVPN. Позволяет добавлять клиентов, скачивать их конфигурационные файлы и отзывать сертификаты.

## Запуск

### Зависимости

```bash
pip install -r requirements.txt
```

### Прямой запуск

```bash
API_KEY=your_secret_key python3 api.py
```

Сервер запустится на `0.0.0.0:8000`.

### Как systemd-сервис

```bash
# Скопировать файл сервиса
cp openvpn-api.service /etc/systemd/system/

# Установить API-ключ в файле сервиса
nano /etc/systemd/system/openvpn-api.service

# Активировать и запустить
systemctl daemon-reload
systemctl enable openvpn-api
systemctl start openvpn-api
```

## Аутентификация

Все запросы требуют заголовка `Authorization` с Bearer-токеном.

```
Authorization: Bearer <API_KEY>
```

Значение `API_KEY` задаётся переменной окружения (по умолчанию `changeme`).

## Эндпоинты

### Добавить пользователя

```
POST /users
```

Создаёт нового клиента OpenVPN и возвращает его `.ovpn`-файл.

**Заголовки:**

| Заголовок      | Значение             |
|----------------|----------------------|
| Authorization  | Bearer `<API_KEY>`   |
| Content-Type   | application/json     |

**Тело запроса:**

```json
{
  "name": "username"
}
```

| Поле | Тип    | Описание                                    |
|------|--------|---------------------------------------------|
| name | string | Имя пользователя (только буквы и цифры)    |

**Ответы:**

| Код | Описание                                      |
|-----|-----------------------------------------------|
| 200 | `.ovpn`-файл (application/octet-stream)       |
| 400 | Поле `name` отсутствует или содержит недопустимые символы |
| 401 | Неверный или отсутствующий токен              |
| 409 | Пользователь с таким именем уже существует    |
| 500 | Ошибка выполнения скрипта или генерации сертификата |

**Пример:**

```bash
curl -X POST http://localhost:8000/users \
  -H "Authorization: Bearer changeme" \
  -H "Content-Type: application/json" \
  -d '{"name": "alice"}' \
  -o alice.ovpn
```

---

### Скачать конфигурацию пользователя

```
GET /users/{name}/cert
```

Возвращает `.ovpn`-файл существующего пользователя.

**Параметры пути:**

| Параметр | Описание           |
|----------|--------------------|
| name     | Имя пользователя   |

**Ответы:**

| Код | Описание                              |
|-----|---------------------------------------|
| 200 | `.ovpn`-файл (application/octet-stream) |
| 401 | Неверный или отсутствующий токен      |
| 404 | Сертификат для пользователя не найден |

**Пример:**

```bash
curl http://localhost:8000/users/alice/cert \
  -H "Authorization: Bearer changeme" \
  -o alice.ovpn
```

---

### Удалить пользователя

```
DELETE /users/{name}
```

Отзывает сертификат пользователя и удаляет его `.ovpn`-файл.

**Параметры пути:**

| Параметр | Описание           |
|----------|--------------------|
| name     | Имя пользователя   |

**Ответы:**

| Код | Описание                              |
|-----|---------------------------------------|
| 204 | Пользователь успешно удалён           |
| 401 | Неверный или отсутствующий токен      |
| 500 | Ошибка выполнения скрипта отзыва      |

**Пример:**

```bash
curl -X DELETE http://localhost:8000/users/alice \
  -H "Authorization: Bearer changeme"
```
